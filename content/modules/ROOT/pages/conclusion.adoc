=== Core Concepts

* **Projects and Namespaces**: Isolation and multi-tenancy
* **Pods**: The smallest deployable units
* **Deployments**: Managing application lifecycles
* **Services**: Stable networking for pods
* **Routes**: External access to applications
* **ConfigMaps and Secrets**: Configuration management

=== Key Skills

* Using the OpenShift web console and CLI (`oc`)
* Deploying applications from container images
* Building applications from source with Source-to-Image (S2I)
* Scaling applications horizontally

== Clean Up Resources

If you want to clean up the resources you created during this workshop:

[source,bash,role=execute,subs="attributes+"]
----
# Delete all deployments
oc delete deployment parksmap nodejs-app -n workshop-{guid}

# Delete all routes
oc delete route --all -n workshop-{guid}

# Delete all services
oc delete service --all -n workshop-{guid}

# Delete all ConfigMaps
oc delete configmap --all -n workshop-{guid}

# Delete all Secrets (except default service account token)
oc delete secret --all -n workshop-{guid}

# (Optional) Delete the entire project
oc delete project workshop-{guid}
----

WARNING: Deleting the project will remove all resources permanently.
