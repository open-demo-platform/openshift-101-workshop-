[#console]
== Exploring the Web Console

The OpenShift web console provides a powerful graphical interface for managing your applications and cluster resources.

=== Accessing the Console

. Open the OpenShift web console in a new browser tab:
+
{openshift_console_url}[OpenShift Console,window=_blank]

. Log in with your credentials:
+
* *Username*: `{user}`
* *Password*: `{password}`

. After logging in, you'll see the **Administrator** perspective by default.

=== Understanding Console Perspectives

OpenShift offers two main perspectives:

* **Administrator**: Full cluster management, operators, projects, and infrastructure
* **Developer**: Application-centric view for building, deploying, and managing apps

. Switch between perspectives using the dropdown in the top-left corner
+
TIP: For this workshop, we'll primarily use the **Administrator** perspective to understand the platform fundamentals.

=== Exploring the Dashboard

. From the **Administrator** perspective, click **Home** → **Overview**
+
You'll see cluster health, resource usage, and recent events.

. Explore the navigation menu on the left:
+
* **Workloads**: Pods, Deployments, StatefulSets
* **Networking**: Services, Routes, Ingresses
* **Storage**: PersistentVolumeClaims, StorageClasses
* **Builds**: BuildConfigs, ImageStreams (OpenShift-specific)
* **Operators**: Operator management and installation

NOTE: The console is context-aware and changes based on the selected project.

[#cli]
== Using the OpenShift CLI

The `oc` command-line tool is the primary way to interact with OpenShift from the terminal.

=== Logging In via CLI

. In the terminal below (or the terminal tab), log in to the OpenShift cluster:
+
[source,bash,role=execute]
----
oc login {openshift_api_url} -u {user} -p {password}
----
+
.Expected Output
[source,textinfo]
----
Login successful.

You have access to 70 projects, the list has been suppressed. You can list all projects with 'oc projects'
----

. Verify your login by checking your current user:
+
[source,bash,role=execute]
----
oc whoami
----
+
.Expected Output
[source,textinfo]
----
{user}
----

. Check the OpenShift server version:
+
[source,bash,role=execute]
----
oc version
----
+
You'll see both client (`oc`) and server (OpenShift) versions.

=== Basic CLI Commands

Let's explore some essential `oc` commands:

. View cluster information:
+
[source,bash,role=execute]
----
oc cluster-info
----

. List all nodes in the cluster:
+
[source,bash,role=execute]
----
oc get nodes
----
+
.Sample Output
[source,textinfo]
----
NAME                                         STATUS   ROLES                  AGE   VERSION
ip-10-0-22-36.us-east-2.compute.internal    Ready    control-plane,master   5d    v1.27.6
ip-10-0-30-58.us-east-2.compute.internal    Ready    worker                 5d    v1.27.6
ip-10-0-34-72.us-east-2.compute.internal    Ready    worker                 5d    v1.27.6
----

. Get more details about a specific node:
+
[source,bash,role=execute]
----
oc describe node $(oc get nodes -o jsonpath='{.items[0].metadata.name}')
----
+
NOTE: This command describes the first node in the cluster. You'll see CPU, memory, pods, and more.

[#projects]
== Projects and Namespaces

In OpenShift, **Projects** are an extension of Kubernetes namespaces with additional features like RBAC and resource quotas.

=== Understanding Projects

Projects provide:

* **Isolation**: Separate resources between teams or applications
* **RBAC**: Role-Based Access Control for who can do what
* **Resource Quotas**: Limits on CPU, memory, and storage
* **Network Policies**: Control traffic between projects

=== Creating Your First Project

. Create a new project for this workshop:
+
[source,bash,role=execute]
----
oc new-project workshop-{guid}
----
+
.Expected Output
[source,textinfo]
----
Now using project "workshop-{guid}" on server "{openshift_api_url}".
----

. Verify you're in the correct project:
+
[source,bash,role=execute]
----
oc project
----
+
.Expected Output
[source,textinfo]
----
Using project "workshop-{guid}" on server "{openshift_api_url}".
----

. List all projects you have access to:
+
[source,bash,role=execute]
----
oc projects
----

=== Exploring Project Details

. Get detailed information about your project:
+
[source,bash,role=execute]
----
oc describe project workshop-{guid}
----
+
You'll see annotations, labels, resource quotas (if any), and more.

. View the project in the web console:
+
.. Go to **Home** → **Projects**
.. Find and click on `workshop-{guid}`
.. Explore the **Overview**, **Workloads**, **YAML**, and **Role Bindings** tabs

=== Switching Between Projects

. List all projects:
+
[source,bash,role=execute]
----
oc get projects
----

. Switch to a different project (e.g., `default`):
+
[source,bash,role=execute]
----
oc project default
----

. Switch back to your workshop project:
+
[source,bash,role=execute]
----
oc project workshop-{guid}
----

TIP: You can use `oc project <project-name>` to switch contexts quickly.

== Understanding Kubernetes Resources in OpenShift

OpenShift is built on Kubernetes, so you can use standard `kubectl` commands with `oc`.

=== Listing Resources

. List all resource types available in the cluster:
+
[source,bash,role=execute]
----
oc api-resources | head -20
----

. List pods in your project (currently none):
+
[source,bash,role=execute]
----
oc get pods
----
+
.Expected Output
[source,textinfo]
----
No resources found in workshop-{guid} namespace.
----

. List pods in all namespaces (admin view):
+
[source,bash,role=execute]
----
oc get pods --all-namespaces | head -10
----
+
You'll see system pods running in various namespaces.

=== Using Labels and Selectors

Labels are key-value pairs attached to objects. They're essential for organizing and selecting resources.

. View all projects with labels:
+
[source,bash,role=execute]
----
oc get projects --show-labels
----

. View only projects with a specific label (if any):
+
[source,bash,role=execute]
----
oc get projects -l "name=workshop-{guid}"
----

== CLI Tips and Tricks

=== Command Shortcuts

The `oc` CLI supports many shortcuts:

* `oc get po` → `oc get pods`
* `oc get svc` → `oc get services`
* `oc get deploy` → `oc get deployments`

. Try getting services with the shortcut:
+
[source,bash,role=execute]
----
oc get svc -n workshop-{guid}
----

=== Output Formats

You can change the output format of `oc` commands:

. Get projects in YAML format:
+
[source,bash,role=execute]
----
oc get project workshop-{guid} -o yaml
----

. Get projects in JSON format:
+
[source,bash,role=execute]
----
oc get project workshop-{guid} -o json
----

. Use JSONPath to extract specific fields:
+
[source,bash,role=execute]
----
oc get project workshop-{guid} -o jsonpath='{.metadata.name}'
----

=== Watching Resources

You can watch resources in real-time using the `-w` flag:

[source,bash]
----
oc get pods -w
----

TIP: Press `Ctrl+C` to stop watching.
